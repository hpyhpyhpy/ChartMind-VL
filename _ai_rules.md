# AI规则

## 1. 任务边界

每次只完成用户指定的任务，不得擅自新增功能。

## 2. 修改要求

每次修改后必须说明：

- 修改/添加了哪些文件
- 为什么修改/添加
- 如何运行/使用
- 如何验收
- 是否影响已有功能

## 3. Git规则

每完成一个可验收任务，必须更新 `project_state.md`，并进行一次 git commit。

## 4. 读取要求

每次新开对话时，AI 必须优先读取本文件，并在后续工作中遵守本文件规则。

## 5. 语言要求

项目中的说明类文档、开发计划、实验记录、README、注释和面向用户的解释，默认使用中文，便于用户快速理解和复盘。

以下内容可保留英文：

- 代码标识符、函数名、类名、变量名
- 第三方库、模型、数据集、论文、工具和命令的官方名称
- 必须与外部接口、配置字段或开源社区约定保持一致的文本
